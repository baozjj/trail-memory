# 任务：实现完整的登录与鉴权模块

我们要开始实现业务逻辑的第一步：用户体系。请按照以下要求，分模块实现登录与注册功能：

## 1. 后端接口实现 (Backend)

请在 `backend/src` 下创建对应的 controller 和 route 文件：

- **数据校验**：使用 `zod` 对所有请求体（email, password 等）进行格式校验。
- **用户存储**：使用 `bcrypt` 对密码进行 hash 加密后再存入 SQLite。
- **安全性**：
  - 登录接口需返回模糊化的报错信息（“邮箱或密码错误”，无论具体是哪项错）。
  - 注册接口中预留 `isVerified` 布尔字段（默认 false），并加入一个逻辑开关：如果环境变量 `EMAIL_VERIFICATION` 为 true 则发送邮件（这里暂时先写注释即可），否则视为注册成功。
- **JWT 鉴权**：
  - 编写一个 `authMiddleware.ts`，验证前端传入的 `Authorization: Bearer <token>`。
  - 在验证成功后，将 `userId` 注入到 `req` 中。
- **接口列表**：
  - `POST /api/auth/register` (注册)
  - `POST /api/auth/login` (登录，返回 JWT)
  - `GET /api/auth/me` (获取当前登录用户信息，需使用鉴权中间件)

## 2. 前端集成 (Frontend)

请在前端项目（`frontend/src`）中完成：

- **API 封装**：在 `src/api` 下创建一个 `auth.ts`，使用 `axios` 封装上述三个接口。
- **状态管理**：
  - 在 `src/utils/auth.ts` 中封装 `getToken`, `setToken`, `removeToken` 方法（操作 localStorage）。
  - 在 `src/api/axios.ts` 中配置 axios 拦截器，自动将存储的 Token 放入请求头的 `Authorization` 字段。
- **页面实现**：
  - 创建 `Login.vue` 和 `Register.vue`，界面风格要求极简，仅包含邮箱、密码输入框和提交按钮。
  - 交互：登录成功后，跳转至首页；若 Token 过期，axios 拦截器应自动重定向至登录页。
