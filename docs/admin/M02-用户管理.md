## 3. 数据模型

```prisma
enum UserStatus {
  ACTIVE
  BANNED
}

model User {
  // 现有字段保留 …
  status UserStatus @default(ACTIVE)
}
```

现有字段参考 `backend/prisma/schema.prisma`：`email`、`nickname`、`signature`、`avatarUrl`、`showCardOnGuestPage`、`isVerified`。

---

## 4. 接口

### GET `/api/admin/users`

Query：`q`、`isVerified`、`status`、`createdFrom`、`createdTo`、`page`、`pageSize`。

响应 `items[]`：

```ts
{
  id: string
  email: string
  nickname: string
  isVerified: boolean
  status: 'ACTIVE' | 'BANNED'
  memoryCount: number      // 未软删印记数
  createdAt: string        // ISO
}
```

### GET `/api/admin/users/:id`

```ts
{
  id, email, nickname, signature, avatarUrl,
  showCardOnGuestPage, isVerified, status,
  createdAt, updatedAt,
  stats: { memoryCount: number, publicMemoryCount: number }
}
```

### PATCH `/api/admin/users/:id`

Body（zod，至少一项）：

```json
{ "status": "BANNED" }
// 或
{ "isVerified": true }
```

审计：

| 变更 | action |
|------|--------|
| → BANNED | `USER_BAN` |
| → ACTIVE | `USER_UNBAN` |
| isVerified true | `USER_VERIFY` |

### GET `/api/admin/users/:id/memories`

分页简表：`id`、`title`、`isPublic`、`createdAt`（仅 `deletedAt null`）。

## 5. 管理端页面

### `/users` 列表

- 筛选：`t-input`（q）、验证状态 `t-select`（status）、注册日期 `t-date-range-picker`
- 列：邮箱、昵称、验证、状态、印记数、注册时间、操作
- 操作：详情；写权限：禁用/启用、标记已验证 → `DialogPlugin.confirm`

### 详情 `t-drawer`

- `t-descriptions` 展示用户字段
- 统计卡片
- 印记简表：`t-table`，「查看全部」→ `/memories?userId=xxx`
