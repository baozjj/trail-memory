## 4. 接口

### GET `/api/admin/memories`

| Query | 说明 |
|-------|------|
| `q` | 标题、`meta` 模糊 |
| `userId` | 所属用户 |
| `userEmail` | 解析为 userId |
| `isPublic` | boolean |
| `typeId` | 印记类型 |
| `hasDeleted` | 默认 `false`；`true` 仅查软删 |
| `createdFrom` / `createdTo` | ISO 日期 |
| `page` / `pageSize` | 分页 |

Item DTO：

```ts
{
  id: string
  title: string
  ownerEmail: string
  ownerNickname: string
  typeId: string | null
  typeLabel: string | null
  isPublic: boolean
  linkSuffix: string
  shareSlug: string          // "{id}-{linkSuffix}"，便于复制
  createdAt: string
  updatedAt: string
  deletedAt: string | null
}
```

### GET `/api/admin/memories/:id`

在列表字段基础上增加：

```ts
{
  content: string
  meta: string
  images: string[]         // JSON 解析后
  coverUrl: string
  heightWeight: number
  user: { id, email, nickname }
}
```

### PATCH `/api/admin/memories/:id/exhibit`

Body：

```json
{ "isPublic": false }
```

MVP **仅允许**改为 `false`（强制下架）；若已是 false 幂等返回 200。

审计：`MEMORY_FORCE_UNPUBLISH`，payload `{ before: { isPublic }, after: { isPublic: false } }`。

### DELETE `/api/admin/memories/:id`

软删：`deletedAt = now()`。已软删再次删除 → 200 幂等。

审计：`MEMORY_SOFT_DELETE`。

## 5. 分享链行为（验收关键）

| 状态 | `GET /api/memories/share/:slug` | C 端主人列表 |
|------|--------------------------------|--------------|
| 正常公开 | 200 | 可见 |
| `isPublic=false` | 404「印记不存在」 | 可见（私密） |
| `deletedAt` 非空 | 404 | 不可见 |

分享 slug 格式：`{memoryId}-{linkSuffix}`，与 `frontend/src/utils/imprint-link.ts` 一致。

管理端复制完整 URL 示例（M08 前可用相对路径）：

```
/m/{id}-{linkSuffix}
```

## 6. 管理端页面

### `/memories`

- 筛选：关键词、用户邮箱、`isPublic`、`typeId`、日期、含已删除 Switch
- 列：标题、用户、类型、公开、分享后缀、封存时间、操作
- 操作：详情 | 强制下架（仅 isPublic） | 删除

### 详情 `t-drawer`

- 基本信息 + `shareSlug` 复制按钮
- `t-image-viewer` / `t-image` 展示 `images`
- 正文可展开段落，默认展示 500 字
- 底部：强制下架、删除
