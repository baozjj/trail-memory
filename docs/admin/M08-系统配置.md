## 3. 数据模型

```prisma
model SystemSetting {
  key       String   @id
  value     String
  updatedAt DateTime @updatedAt
  updatedBy String?
}
```

### 预置 key

| key | 类型 | env 默认 | 说明 |
|-----|------|----------|------|
| `share.domain` | string | — | 完整域名，如 `https://trail.example.com`（无尾斜杠） |
| `auth.emailVerification` | boolean | `EMAIL_VERIFICATION` env | 新注册是否必须验证邮箱 |

读取逻辑：

```ts
async function getSetting(key: string): Promise<string> {
  const row = await prisma.systemSetting.findUnique({ where: { key } })
  if (row) return row.value
  return getEnvFallback(key)  // 映射 env
}
```

## 4. 接口

### GET `/api/admin/settings`

需 `settings.read`。返回：

```json
{
  "success": true,
  "data": {
    "items": [
      { "key": "share.domain", "value": "https://trail.example.com", "updatedAt": "...", "updatedBy": "..." },
      { "key": "auth.emailVerification", "value": "false" }
    ]
  }
}
```

### PATCH `/api/admin/settings`

需 `settings.write`（**仅 SUPER_ADMIN**）。

```json
{
  "items": [
    { "key": "share.domain", "value": "https://example.com" },
    { "key": "auth.emailVerification", "value": "true" }
  ]
}
```

校验：

- `share.domain`：合法 URL，https 开头
- `auth.emailVerification`：`"true"` | `"false"`

审计：`SETTINGS_UPDATE`，payload 含变更前后快照。

### GET `/api/settings/public`（C 端）

```json
{
  "success": true,
  "data": {
    "shareDomain": "https://trail.example.com"
  }
}
```

不暴露 `auth.emailVerification`。
