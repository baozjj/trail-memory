## 1. 模块目标

只读浏览本地上传目录；展示文件是否被印记/头像引用；**不提供删除**。

## 3. 接口

### GET `/api/admin/media/stats`

```json
{
  "success": true,
  "data": {
    "fileCount": 120,
    "totalBytes": 45678901
  }
}
```

### GET `/api/admin/media`

Query：`page`、`pageSize`、`q`（文件名模糊，可选）。

```json
{
  "success": true,
  "data": {
    "items": [
      {
        "filename": "abc.jpg",
        "url": "/uploads/abc.jpg",
        "sizeBytes": 102400,
        "mtime": "2026-05-20T10:00:00.000Z",
        "referenced": true,
        "referenceCount": 2
      }
    ],
    "total": 120,
    "page": 1,
    "pageSize": 20
  }
}
```

### GET `/api/admin/media/:filename/references`

```json
{
  "success": true,
  "data": {
    "memories": [{ "id": "...", "title": "..." }],
    "users": [{ "id": "...", "email": "..." }]
  }
}
```

**引用检测实现**：

- `Memory.images` 字符串包含 `/uploads/{filename}`
- `User.avatarUrl === '/uploads/{filename}'` 或包含该路径
- 注意 URL 编码文件名边缘情况（MVP 可只处理原始文件名）

---

## 4. 管理端页面 `/media`

- 顶部指标：`t-card` 展示文件数、总 MB（`totalBytes / 1024 / 1024` 保留 1 位）
- `t-table`：缩略图列用 `t-image`（仅 image/*）、文件名、大小、修改时间、是否引用
- 点击行：右侧 `t-drawer` 列出引用印记（链接 `/memories` 带筛选）与用户
